Greetings and salutations, thank you for deciding to learn more about web development and taking the extra mile to learn Unix and Git. I assure you that having these tools in your belt will make your future in Web development much easier.

So what is Git? Git is two things, an internet protocol and Version-Control Software. Git has become very popular among the open source community for its sophisticated branching and merging abilities as well as being commandline friendly. Allowing it to be software and platform independent

In this tutorial I will introduce you to the fundamental commands for Unix and Git as well as step you through how to install these technologies on your respected platforms

  • Linux
  • Mac
  • Windows

Warning: This tutorial was written on Ubuntu 14.04 LTS and Git 1.9.1. I appologize if you come across any compatibility issues.

Step 1: Install Git

First thing first open a terminal window 

keyboard: alt+ctrl+T
or 
hit SUPER then type in Terminal

Yours should look something like this

Next you should install Git. Type the following into your terminal 

sudo apt-get update && sudo apt-get install git

You should be prompted with a Sudo request. This is your admin password

Next type the following into your terminal to double check it was successfully installed 

git version
If you get something along the lines of "git is unrecognizable function" Then something went wrong. run the command above again or consult the official git site

Step 2: Learn a lil Unix

Next thing is we need to give you a foundation for nativaging in the unix shell.

Note! when you see $ before a string. This is just to express this is a unix command. For example $ sudo apt-get install git you are supposed to type 'sudo apt-get install git' into terminal.

ls

The command ls is used to list all files and folders at the current level

mkdir

mkdir is short for 'make directory' so you use this command to make new folders. Example: $ mkdir test_folder. This will make a new folder at the current level called test_folder. Try it out for yourself and then type $ ls

cd

cd is short for 'Change Directory' so you use this command to navigate into and out of folers. Example: $mkdir test && cd test this will make a new folder called test and then put you into test. The && is just used to combine the two commands into one line. You can chain all kind of terminal commands together.

../

while technically not a command, this is a way to represent the higher tier. You use this if you want to move the path up a level. For example: You make a folder called test and then you cded into it. Now you want to move back up to the original location. you would use $ cd ../ Try it out for yourself

If you want to learn move about these commands I recommend checking out the documentation at linfo.org

Part 3: Lets write some code

two more unix commands

touch

touch is similiar to mkdir. Except instead of making new folders it is constructing a new file. For example $ touch helloworld.rb This will create a new blank ruby script file called helloworld

nano

nano is a built in terminal editor. There are many other ones but I prefer nano because of its simple Write function which I will demonstrate later. to use nano just type $ nano file_name.extension. Example: $ nano helloworld.rb
Now lets make a program :D type in $ touch helloworld.rb and then type in $ nano helloworld.rb Then type in the following 
						
puts 'hello world'
Should look like something like this then type ctrl+o to save it followed with ctrl+x to exit

next type in $ ruby helloworld.rb This will tell ruby to run this new script and it should print out 'hello world'

if you recieve message like "ruby is an unrecognized function" then you are on a version of Linux that doesn't have Ruby preinstalled. you can install it with $ sudo apt-get install ruby or instal Ruby Version Manager

Part 4: Git started

Now we have a program lets use some version control. Type in the following 4 commands in sequence

git init
git add -A
git status
git commit -m "initialize"

Does yours look like mine??

Lets break it down line by line

git init This is initializing an empty git repo at the current location

git add -A git add adds files to the git repo. For example, $ git add helloworld.rb would also have added our file. However, we used the suffix -A instead because it searches and finds all valid options and adds them.

git status Status states whats current on the stage. This step isn't necessary but I want to exspose some of the internal working of git. You can think of it like this: In a git repo, a file has 3 states [ indepenent, on stage , committed]. A commited file is a file with a history in the repo, an independent file is not a part of the repo. Either it hasn't been added to or it demeed unneceasry anymore and was removed from the repo. The final one is the stage. This is what status tells us, the stage is important because these are the files of interest for this commit. Its how git determines what files are going to cause merge conflicts. It'll be come clearer the more we play with git.

git commit -m "initialize" Commits are the cornerstone of git. Commits are the nodes in the branching graphs in Git. For those unfamiliar with graph thoery, commits are the different version builds they are solid merged states. Git can revert back to any git commit. Each on of those dots are a different commit. The current build is at dev but you could move the program back to a previous node like feature_x and change all of ur current files back to that old state.

Part 5: Do some branching

Hey so we have a git repo made with an initial node....why dont we start playing around with this :D

open helloworld.rb up again with nano

$ nano helloworld.rb

	
puts "hello world"
puts "My name is YOUR_NAME"

Now save it and run it with $ ruby helloworld.rb It should have printed out both lines

Now lets save your progress with

git add -A
git commit -m "added one more line"

Now your git is up to date. However...what if we wanted to undo this change? well one way is to move back a commit

We accomplish this by typing in the following

git reset --hard HEAD~1
Note: That is HEAD~1 with a tilde not a minus sign

Now that should have removed our changes and reverted us back to the old version. You can check this out by typing in $ ruby helloworld.rb. While this solution works its like killing a fly with a bazooka

Now we can always just edit what we want to undo and just make a new commit. However when handling thousands of files...that option isn't a small task. So why don't we employ a more sophisticated solution to a making a minor change.

open Hello world up again using nano $ nano helloworld.rb and copy the following: 

 

x = 0
x += 5
puts "Hello I am Android#{x}"

Now add and commit. Ex: $ git add -A && git commit -m "Android5"

Now type in $git show. This shows you the commit message as well as showing you the different changes made at this level. You can go and manual undo those changes

Next lets do some branching. Branches are useful for when you want to develop a particular feature or make a major change but don't know how it will affect the rest of the system. So you create a seperate version just for yourself to experiement. Then when you have solved all the issues you can merge it with the master

git branch foo_branch
git checkout foo_branch

$ git branch BRANCH_NAME makes a new git branch

$ git checkout BRANCH_NAME moves the HEAD pointer to this named branch

You can actually combine the two commands into $ git checkout -b BRANCH_NAME

Now remove everything in helloworld.rb and copy this:


class Foo
  attr_accessor :bar
  def initialize(b)
    @bar = b
  end
end

f = Foo.new('FuzzBuzz')
puts f.bar

Now lets add these changes and commit them. $ git commit -a -m "added Foo class"

When you run $ruby helloworld.rb you will notice it only says FuzzBuzz

However, if you type in $git checkout master and run $ruby helloworld.rb it should print out Hello, I am Android5

Part 6: Merging

The next major topic is merging. Merging allows you to combine different branches together. I will demonstrate how to use git's tools to handle merge conflictions.

Now lets make sure our head is at master $git checkout master now lets change the code to helloworld.rb


puts "Master += foo_branch"

now add and commit $git commit -a -m "Master += foo_branch"

run $ ruby helloworld.rb

okay now run $ git merge foo_branch

You should get this message

$ git merge foo_branch
Auto-merging helloworld.rb
CONFLICT (content): Merge conflict in helloworld.rb
Automatic merge failed; fix conflicts and then commit the result.

So we now need to clean up and resolve the merge issues before we can proceed. Type in $git status

It will state we have merge issues. So we need to resolve this. We can manually go through our files and find the instances where there are <===HEAD== like comments. Unforunately we could easily miss one going that route. Instead we will use git's mergetools

Type in $git mergetools and then enter

This will bring up the gvimdiff tool. This program is built upon vim is very complex. Vim itself could fill up a tutorial or two. However, do not fear, we can easily use this program without any VIM knowledge. Instead we will just copy and paste the snippets we want.

The structure of these 4 windows is as such:

| Local | Base | Remote |
|        Merge          |

so Local is the current branch we are at the end of master

Remote is the branch we are merging with so Foo_branch

Base is the last commit these two branches have in common and Merge is the result we want. Just edit Merge how you want.

Note VIM has multiple states which can be confusing at first. You want to be in the INSERT mode for this tutorial. To get into that state just press ESC and then press I . Now you will be able to type in where you put your cursor. You can just type in what you want into Merge or you may copy and paste from Local and Remote. I recommend using the GUI buttons. The keyboard commands vary.

The merged result should look like this:


puts "Master += foo_branch"

class Foo
  attr_accessor :bar
  def init(b)
    @bar = b
  end
end

f = Foo.new("FuzzBuzz")
puts f.bar

Now save and quit this program

run $ git commit -a -m "Fixed merge conflicts"

and run $ ruby helloworld.rb again.

Master += foo_branch
FuzzBuzz

If that was your output, then congratulations you have successfully merged the two branches. The only thing left is to host it online
